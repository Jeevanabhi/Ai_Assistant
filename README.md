# ECI Voting Assistant

A smart, dynamic AI assistant designed to help Indian citizens navigate the election process, registration timelines, and necessary forms in an interactive and easy-to-follow way.

## 🎯 Chosen Vertical
**Civic Technology & Public Utility**
We chose this vertical to address the critical need for accessible, accurate, and personalized voting information. Navigating government portals can be overwhelming; this assistant bridges the gap by providing conversational, tailored guidance based strictly on official data.

## 🧠 Approach and Logic
Our core approach centers around **Factual Accuracy** and **Context-Aware Logic**.

1. **Strict Guardrails**: To prevent AI hallucinations or misinformation, the Google Gemini API is constrained by a strict System Prompt. It is instructed that its *only* source of truth is `https://voters.eci.gov.in/`. If it does not know an answer, it is hard-coded to admit ignorance rather than guess.
2. **Deterministic Output**: The AI's `temperature` parameter is set to `0.1` to ensure highly factual, predictable, and logical responses.
3. **Dynamic User Context**: Rather than providing generic advice, the application implements a "Voter Profile" state. By selecting a profile (e.g., "First-Time Voter" or "NRI Voter"), the application injects this context directly into the AI's logic engine. The AI logically decides which information is relevant based on this context (e.g., recommending Form 6A exclusively to NRIs).
4. **Security & Performance**: The React frontend is heavily optimized using `useCallback` and `React.memo` to prevent unnecessary re-renders. Furthermore, all Markdown output generated by the AI is strictly sanitized using `DOMPurify` to eliminate Cross-Site Scripting (XSS) vulnerabilities.

## ⚙️ How the Solution Works
1. **Context Selection**: The user selects their specific demographic profile from the dropdown in the header.
2. **User Input**: The user asks a question in the chat interface (e.g., "How do I register?").
3. **Payload Construction**: The application bundles the user's message, the chat history, and the user's profile context into a specialized payload. It strips out local UI-only messages to ensure the API receives perfectly formatted historical data.
4. **Google Services Integration**: The payload is sent to the **Google Gemini API** (`gemini-flash-latest`). 
5. **Sanitization & Rendering**: The AI responds with structured Markdown (bullet points, bold text). The application intercepts this response, runs it through `DOMPurify` for security, and renders it beautifully into the glassmorphism UI.

## 🤔 Assumptions Made
* **Source of Truth**: We assume that `https://voters.eci.gov.in/` contains the most up-to-date and legally accurate information regarding Indian elections.
* **Connectivity**: We assume the user has a stable internet connection to communicate with the Google Gemini API.
* **Accessibility**: We assume the user might rely on screen readers or keyboard navigation, which is why the application is built with strict ARIA labels (`aria-live`, `aria-label`) and high-contrast CSS.
* **Model Capability**: We assume the `gemini-flash-latest` model provides the optimal balance of reasoning capabilities, context-window size, and low-latency response times for real-time chat applications.

## 🚀 Getting Started
1. Clone the repository.
2. Run `npm install` to install dependencies (including `dompurify`, `@google/genai`, and `vitest`).
3. Create a `.env` file in the root directory and add your Google Gemini API key: `VITE_GEMINI_API_KEY=your_key_here`
4. Run `npm run dev` to start the local Vite server.
5. Run `npm test` to execute the Vitest testing suite.
